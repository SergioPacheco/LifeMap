## Regras de Ouro para Relatórios

1. **Nunca diagnosticar** — nem saúde, nem psicologia, nem destino
2. **Sempre incluir agência** — "tendência a", "convite para", "potencial de"
3. **Mapa Infantil: zero rótulos negativos** — usar linguagem de potencial
4. **Compatibilidade: nunca declarar impossibilidade** — todo mapa tem pontos de apoio
5. **Saúde: apenas correlações simbólicas** — nunca prescrever ou diagnosticar
6. **Financeiro: não prometer riqueza** — mostrar talentos monetizáveis e ciclos
7. **Predições: linguagem de ciclo** — "o período favorece", não "vai acontecer"

## Arquitetura de Extensão

Para criar um novo relatório:

1. Definir quais **dados astrológicos** são necessários (já calculados pelo engine)
2. Definir as **regras interpretativas** específicas (quais planetas/casas/aspectos priorizar)
3. Criar função `generateXxxPdf()` em `report-generators.ts`
4. Adicionar textos interpretativos em `src/engine/interpretations/{locale}.ts`
5. Adicionar produto no `PRODUCTS` de `payment.ts` e `ReportsShop.tsx`
6. Criar página `reports/xxx-sample.astro`

O motor é o mesmo — o que muda é:
- **Quais regras têm prioridade** (cada relatório pesa diferentes planetas/casas)
- **Qual linguagem usar** (infantil = positiva, psicológico = profunda, financeiro = prática)
- **Quais seções incluir** (cada tipo tem estrutura própria)
